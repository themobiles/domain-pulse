# Domain Pulse | ドメイン確認 & 接続情報ツール

[👉 Click here for English README (英語版のREADME)](./README.md)

新しいドメインの取得時やメンテナンス時に、そのドメインが現在有効（オンライン）であることをスタイリッシュに伝えるためのプレースホルダー用テンプレートです。

訪問者のネットワーク接続情報（IPアドレス、プロバイダ/ASN、現在地、ブラウザ、プロトコル、暗号化方式）を美しく表示する「簡易確認くん」ツールを標準搭載しています。

**Cloudflare Pages & Functions** を利用し、サーバーレスかつ完全無料、外部APIへの依存なしで動作します。

---

## 🚀 特徴

- **ミニマル・モダンデザイン**: 派手なグラデーションや重いガラスモーフィズムを廃し、`Inter` フォントをベースとしたタイポグラフィ主体のクリーンなスイスモダン風UI。ステータスの細かな点滅（ping）やプログレスバーの微細なシマー（揺らぎ）といった洗練されたマイクロアニメーションを採用。
- **多言語対応 (日本語/英語)**: ブラウザの言語設定を自動で検出し、適切な言語にUIをローカライズします。
- **ドメイン名の自動取得**: ドメイン名は一切ハードコードされていません。JavaScriptが現在のアドレス (`window.location.hostname`) から動的に取得してヘッダーやフッター、タイトルを生成します。どのドメインでもそのまま使えます。
- **サーバーレスAPI**: 外部サービスに依存せず、Cloudflare Edgeサーバー上で直接ヘッダー解析を行い、以下の情報を取得します。
  - 接続元IPアドレス (IPv4 / IPv6)
  - 接続プロバイダ (ISP) / 自律システム番号 (ASN / 組織名)
  - 接続元の国・地域・都市 (CloudflareのGeoIP)
  - 使用ブラウザ (User-Agent)
- **インフラオタク向け詳細診断 (Advanced Diagnostics)**:
  - **エッジサーバーの空港コード逆引き**: `CF-RAY` ヘッダーの末尾に含まれるIATA（空港）コード（例: `NRT`, `HND`, `KIX`）から、実際に通信を処理している最寄りのデータセンターの都市名（例: `東京 / 羽田 (HND)`) に自動変換して可視化します。
  - **TLS 暗号スイートの確認**: ブラウザとエッジ間で合意された暗号化アルゴリズム（例: `AEAD-AES256-GCM-SHA384`）を表示します。
  - **TLS バージョン & プロトコル**: 接続されたプロトコル (例: `HTTP/3 (QUIC)`) と TLS バージョンを表示します。
  - **RTT (応答速度) 計測**: クライアントとエッジ間の往復時間（ミリ秒）をリアルタイムで測定します。
  - **IPv6 アドレスの完全展開**: IPv6で接続された場合、ゼロ省略（`::`）されたアドレスをすべてゼロパディングしたフル表記の128ビット（例: `240d:000f:0942:6500:2529:6aa9:28f3:b254`）に展開表示します。
  - **IPv4 並行検出**: IPv6接続時に、バックグラウンドで `ipv4.icanhazip.com` を呼び出し、自身のパブリックIPv4アドレスも並行して取得・表示します。
  - **DNSSEC 署名検証**: Cloudflareの DoH (DNS over HTTPS) API をバックグラウンドで呼び出し、アクセスしたドメインが DNSSEC（電子署名）で保護され、リゾルバで署名検証（ADフラグ検知）されているかを確認します。
  - **Alt-Svc パラメータ追跡**: サーバーから送られる HTTP/3 移行用の Alt-Svc ヘッダー値を抽出し、表示します。
  - **HTTP 多重化 (Multiplexing) 接続診断**: リソースタイミングAPIを解析し、全リソースが同一の TCP/UDP コネクション上で多重化（Concurrent Streams）されて処理されているかを可視化します。

## ☁️ Cloudflare Pages へのデプロイ手順

このテンプレートは GitHub と Cloudflare Pages を連携させることで、`git push` するだけで自動デプロイが完了します。

### 1. Cloudflare Pages プロジェクトの作成
1. **Cloudflare ダッシュボード**にログインします。
2. 左メニューの **「Workers と Pages」** ＞ **「概要」** ＞ **「作成」** をクリックします。
3. **「Pages」** タブ（Workersの右隣）を選択し、**「Git に接続」** をクリックします。
4. GitHubアカウントと連携し、本リポジトリ (`domain-pulse`) を選択します。
5. ビルド設定画面では、以下のように設定します（基本はデフォルトのままで動作します）。
   - **フレームワーク プリセット**: `なし`（または `静的HTML`）
   - **ビルドコマンド**: (空欄のまま)
   - **出力ディレクトリ**: `.` (ルートを示すドット1つ、または空欄)
6. **「保存してデプロイ」** をクリックします。

### 2. カスタムドメインの設定
1. デプロイ完了後、Pagesプロジェクト管理画面の **「カスタムドメイン」** タブに移動します。
2. **「カスタムドメインの設定」** をクリックし、お持ちのドメイン (例: `yourdomain.com`) を追加します。
3. 画面の指示に従ってDNSレコードを設定（または自動承認）します。
4. 必要に応じて、`www.yourdomain.com` も同様に追加します。

## 🛠️ ローカル環境での開発とテスト

APIを含めた動作確認をローカルで行うには、Cloudflare公式の CLI ツール `wrangler` を使用します。

1. **ローカルサーバーの起動**:
   ```bash
   npx wrangler pages dev .
   ```
   *(初回実行時は Wrangler CLI のインストール確認が表示されます。)*

2. **ブラウザでアクセス**:
   - `http://localhost:8788` にアクセスすると、ローカル用のデモデータが注入された状態でフロントエンドが表示されます。
   - `http://localhost:8788/api/info` で API自体のJSON出力を確認できます。

## 🔒 セキュリティ & プライバシーベストプラクティス

- **クレデンシャルの非保持**: 本プロジェクトは外部APIキーなどを使いません。絶対にパスワードやAPIトークンなどのファイルをプロジェクト内に作成してコミットしないでください。
- **キャッシュ制御とプライバシー保護**: `/api/info` APIはレスポンスに `Cache-Control: no-store, no-cache, must-revalidate` を明示し、接続情報がエッジやブラウザに誤ってキャッシュされるのを防ぎ、共有ネットワーク上での他人の個人情報漏洩を防止します。
- **CORS 制限の厳格化**: APIレスポンスは同一オリジン（Same-Origin）からのみの要求を受け付けるよう制限されており、無許可の外部サイトからのAPI情報取得を防止します。
- **Git除外の維持**: `.gitignore` にはWranglerの一時キャッシュファイル (`.wrangler/`) やローカル環境設定ファイル (`.dev.vars`) が登録されています。これらを誤ってGitの管理対象に含めないようご注意ください。
